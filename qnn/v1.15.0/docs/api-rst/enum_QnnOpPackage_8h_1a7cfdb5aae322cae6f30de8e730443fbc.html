enum QnnOpPackage_Error_t¶

QNN OpPackage API result / error codes.

Values:

QNN_OP_PACKAGE_MIN_ERROR = 3000¶
QNN_OP_PACKAGE_NO_ERROR = 0¶
QNN_OP_PACKAGE_ERROR_UNSUPPORTED_FEATURE = QNN_COMMON_ERROR_NOT_SUPPORTED¶

There is optional API component that is not supported yet. See QnnProperty.

QNN_OP_PACKAGE_ERROR_LIBRARY_ALREADY_INITIALIZED = 3000 + 0¶

Indicates op package library was already initialized.

QNN_OP_PACKAGE_ERROR_LIBRARY_NOT_INITIALIZED = 3000 + 1¶

Attempt was made to call a function in an uninitialized op package library. Unless otherwise noted, any op package function may return this error.

QNN_OP_PACKAGE_ERROR_INVALID_INFRASTRUCTURE = 3000 + 100¶

Invalid infrastructure object used in initializing op package.

QNN_OP_PACKAGE_ERROR_INVALID_INFO = 3000 + 101¶

Invalid op package info object used in initializing op package.

QNN_OP_PACKAGE_ERROR_VALIDATION_FAILURE = 3000 + 110¶

Op configuration failed validation.

QNN_OP_PACKAGE_ERROR_INVALID_ARGUMENT = 3000 + 200¶

Invalid function argument.

QNN_OP_PACKAGE_ERROR_GENERAL = QNN_COMMON_ERROR_GENERAL¶

Indicates an error has occurred due to a condition unforeseen by QNN, and possibly meaningful only in the context of the particular op package. Unless otherwise noted, any op package function may return this error.

QNN_OP_PACKAGE_MAX_ERROR = 3999¶
QNN_OP_PACKAGE_ERROR_UNDEFINED = 0x7FFFFFFF¶
