Function QnnTensor_createGraphTensor¶

  • Defined in File QnnTensor.h

Function Documentation¶

Qnn_ErrorHandle_t QnnTensor_createGraphTensor(Qnn_GraphHandle_t graph, const Qnn_Tensor_t tensor)

A function to create a new tensor on Qnn_GraphHandle_t.

This call may or may not allocate memory, depending on the Qnn_TensorType_t value specified in tensor and the accelerator implementation. Optionally it may be initialized with data provided in the tensor if present.

Parameters

  • [in] graph: A handler to the graph which owns this tensor.

  • [in] tensor: A struct containing information on the tensor (type, ID, data format, dimensions, data, etc). For tensors containing static data (such as weights or biases), the tensor type is expected to be QNN_TENSOR_TYPE_STATIC. Valid data must be presented in the tensor object at creation. This data will be copied, and may be safely de-allocated after this call returns. Other tensor types (e.g: NATIVE, APP_READ, APP_WRITE, NULL) must have the data pointer set to NULL at the time of creation. Tensor ID is the key to identify a tensor and therefore, must be unique in the context within which the tensor is created. Furthermore, tensor IDs used with tensors stored in an offline-prepared context binary must be identical to those created at runtime for execution.

Warning

Graph tensors cannot be of type QNN_TENSOR_TYPE_APP_READWRITE. R/W tensors connect multiple graphs.

Return

Error code:

  • QNN_SUCCESS: Successfully created a graph tensor

  • QNN_TENSOR_ERROR_INVALID_HANDLE: Provided graph handle is invalid

  • QNN_TENSOR_ERROR_INVALID_TENSOR_PARAM: One or more tensor parameters is invalid

  • QNN_COMMON_ERROR_MEM_ALLOC: Failure in creating tensor due to issues with memory allocation.

Note

DEPRECATED: Use corresponding API through QnnInterface_t.
