Error Codes¶

This page summarizes QNN API error codes.

QNN_SUCCESS 0

Global value indicating success.

enum QnnCommon_Error_t

An enum which defines error codes commonly used across API components.

Values:

QNN_COMMON_MIN_ERROR = 1000
QNN_COMMON_ERROR_NOT_SUPPORTED = 1000 + 0

Indicates an API or a feature is not supported by implementation. Generally applicable to optional elements of the API.

QNN_COMMON_ERROR_BACKEND_NOT_INITIALIZED = 1000 + 1

Indicates Backend library has not been initialized (see QnnBackend_initialize).

QNN_COMMON_ERROR_MEM_ALLOC = 1000 + 2

Indicates memory allocation related error.

QNN_COMMON_ERROR_SYSTEM = 1000 + 3

Indicates system level error, such as related to platform / OS services.

QNN_COMMON_ERROR_INVALID_ARGUMENT = 1000 + 4

Indicates invalid function argument.

QNN_COMMON_ERROR_OPERATION_NOT_PERMITTED = 1000 + 5

Indicates an illegal operation or sequence of operations.

QNN_COMMON_ERROR_PLATFORM_NOT_SUPPORTED = 1000 + 6

Indicates failure in attempting to use QNN API on an unsupported platform.

QNN_COMMON_ERROR_SYSTEM_COMMUNICATION = 1000 + 7

Communication errors with platform / OS service.

QNN_COMMON_ERROR_GENERAL = 1000 + 100

Indicates general type of error, which has not been identified as any other error type. In general, this error should rarely be used.

QNN_COMMON_MAX_ERROR = 1999
QNN_COMMON_ERROR_UNDEFINED = 0x7FFFFFFF
enum QnnProperty_Error_t

QNN Property API result / error codes.

Values:

QNN_PROPERTY_MIN_ERROR = 2000
QNN_PROPERTY_NO_ERROR = 0
QNN_PROPERTY_SUPPORTED = 0

Property in question is supported.

QNN_PROPERTY_NOT_SUPPORTED = QNN_COMMON_ERROR_NOT_SUPPORTED

Property in question not supported.

QNN_PROPERTY_ERROR_UNKNOWN_KEY = 2000 + 0

The the property key was not known to a backend.

QNN_PROPERTY_ERROR_WRONG_KEY_TYPE = 2000 + 1

The property key refers to a different property type (capability vs property) than is queried via the called function.

QNN_PROPERTY_ERROR_GENERAL = 2000 + 2

An unknown error occurred while attempting to process the property query. This likely indicates the backend is non-functional.

QNN_PROPERTY_ERROR_BAD_FREE = 2000 + 3

Attempt to free a pointer which is not recognized as a value pointer previously returned by a QnnProperty_get call on this backend.

QNN_PROPERTY_MAX_ERROR = 2999
QNN_PROPERTY_ERROR_UNDEFINED = 0x7FFFFFFF
enum QnnOpPackage_Error_t

QNN OpPackage API result / error codes.

Values:

QNN_OP_PACKAGE_MIN_ERROR = 3000
QNN_OP_PACKAGE_NO_ERROR = 0
QNN_OP_PACKAGE_ERROR_UNSUPPORTED_FEATURE = QNN_COMMON_ERROR_NOT_SUPPORTED

There is optional API component that is not supported yet. See QnnProperty.

QNN_OP_PACKAGE_ERROR_LIBRARY_ALREADY_INITIALIZED = 3000 + 0

Indicates op package library was already initialized.

QNN_OP_PACKAGE_ERROR_LIBRARY_NOT_INITIALIZED = 3000 + 1

Attempt was made to call a function in an uninitialized op package library. Unless otherwise noted, any op package function may return this error.

QNN_OP_PACKAGE_ERROR_INVALID_INFRASTRUCTURE = 3000 + 100

Invalid infrastructure object used in initializing op package.

QNN_OP_PACKAGE_ERROR_INVALID_INFO = 3000 + 101

Invalid op package info object used in initializing op package.

QNN_OP_PACKAGE_ERROR_VALIDATION_FAILURE = 3000 + 110

Op configuration failed validation.

QNN_OP_PACKAGE_ERROR_INVALID_ARGUMENT = 3000 + 200

Invalid function argument.

QNN_OP_PACKAGE_ERROR_GENERAL = QNN_COMMON_ERROR_GENERAL

Indicates an error has occurred due to a condition unforeseen by QNN, and possibly meaningful only in the context of the particular op package. Unless otherwise noted, any op package function may return this error.

QNN_OP_PACKAGE_MAX_ERROR = 3999
QNN_OP_PACKAGE_ERROR_UNDEFINED = 0x7FFFFFFF
