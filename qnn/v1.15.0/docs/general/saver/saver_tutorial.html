QNN Saver Tutorial: Save execution sequence with Saver and replay on a backend¶

This tutorial describes how to use the QNN Saver backend to capture the execution sequence of QNN APIs. The captured output can then be compiled and replayed on any QNN backend.

Generating saver_output.c¶

saver_output.c is an artifact produced by running a model on the Saver backend. A model in a source framework can be converted into a model.cpp and model.bin using the QNN Converters. Sample model.cpp and model.bin files are located in <QNN_SDK_ROOT>/examples/converter/models/.

model.cpp and model.bin are used to create a model.so via qnn-model-lib-generator.

$ ${QNN_SDK_ROOT}/target/x86_64-linux-clang/bin/qnn-model-lib-generator \
              -c ${QNN_SDK_ROOT}/examples/converter/models/qnn_model_float.cpp \
              -b ${QNN_SDK_ROOT}/examples/converter/models/qnn_model_float.bin \
              -o ${QNN_SDK_ROOT}/examples/converter/model_libs # This can be any path

This will produce the following artifacts:

  • ${QNN_SDK_ROOT}/examples/converter/model_libs/aarch64-android/libqnn_model_float.so

  • ${QNN_SDK_ROOT}/examples/converter/model_libs/arm-android/libqnn_model_float.so

  • ${QNN_SDK_ROOT}/examples/converter/model_libs/x86_64-linux-clang/libqnn_model_float.so

The resulting model.so can be run on the Saver backend using qnn-net-run:

$ cd ${QNN_SDK_ROOT}/examples/converter/models/
$ ${QNN_SDK_ROOT}/target/x86_64-linux-clang/bin/qnn-net-run \
              --backend ${QNN_SDK_ROOT}/target/x86_64-linux-clang/lib/libQnnSaver.so \
              --model ${QNN_SDK_ROOT}/examples/converter/model_libs/x86_64-linux-clang/model.so \
              --input_list input_list_float.txt

This will produce the following artifacts:

  • ./saver_output/saver_output.c

  • ./saver_output/params.bin

See Tutorial #2 for more information on converting a model from a source framework into the QNN shared library format.

Alternatively, the following script performs the steps described above and can be used for convenience.

$ cd ${QNN_SDK_ROOT}/examples/NetRun/linux-x86_64
$ ./linux-qnn-net-run.sh -b saver

This will produce a quantized saver_output.c and params.bin in <QNN_SDK_ROOT>/examples/NetRun/linux-x86_64/saver_output/quantized/ for replay on DSP or HTP backends, and a non-quantized saver_output.c and params.bin in <QNN_SDK_ROOT>/examples/NetRun/linux-x86_64/saver_output/non_quantized/ for replay on CPU or GPU backends.

Compiling saver_output.c¶

To compile saver_output.c for replay on a backend, use the Makefile located in ${QNN_SDK_ROOT}/examples/Saver. This makefile can be used to compile saver_output.c for the specified QNN backend on a compatible platform.

Note

  • Compiling for x86 targets requires clang.

  • Compiling for Android targets requires Android NDK.

  • Compiling for DSP/HTP backends on Android aarch64 targets requires Hexagon SDK.

Refer to Setup to set the appropriate environment variables.

With saver_output.c located in the same directory as Makefile, run make all to compile for all QNN backends on supported targets.

$ cp ./saver_output/saver_output.c ${QNN_SDK_ROOT}/examples/Saver
$ cp ./saver_output/params.bin ${QNN_SDK_ROOT}/examples/Saver
$ cd ${QNN_SDK_ROOT}/examples/Saver
$ make all

This will produce a folder for each target:

  • ./x86_64-linux-clang

  • ./arm-android

  • ./aarch64-android

These folders will contain one executable for each backend available on the target named saver_output_<backend>.

To create executables for all backends on a specific target, run make <target>. Valid options for <target> are x86, aarch64, or arm.

To create an executable for a specific backend on a specific target, run make <backend>_<target>. Valid options for <backend> are cpu, or htp for <target>=x86, and cpu, gpu, dsp, or htp for <target>=arm or aarch64.

Replaying saver_output.c on x86¶

To replay on x86 backends, set LD_LIBRARY_PATH to point to the desired QNN backend library and run the executable. Ensure params.bin is in the current working directory.

$ cd ${QNN_SDK_ROOT}/examples/Saver
$ export QNN_BACKEND=<backend> # where <backend> is QnnCpu or QnnHtp
$ export LD_LIBRARY_PATH=${QNN_SDK_ROOT}/target/x86_64-linux-clang/lib:$LD_LIBRARY_PATH
$ ./x86_64-linux-clang/saver_output_${QNN_BACKEND} [--logging <verbose,debug,info,warn,error>] # optionally enable logging

Replaying saver_output.c on Android¶

Set environment variables based on platform¶

Set QNN_TARGET_ARCH to indicate the target platform ABI.

$ export QNN_TARGET_ARCH=<target_arch> # where <target_arch> is arm-android or aarch64-android

Replay on QNN CPU or QNN GPU Backend¶

Make a directory on device, push QNN backend library, saver_output.c executable, and params.bin:

$ export QNN_BACKEND=<backend> # where <backend> is QnnCpu or QnnGpu
$ adb shell "mkdir /data/local/tmp/saver"
$ adb push ${QNN_SDK_ROOT}/target/$QNN_TARGET_ARCH/lib/lib${QNN_BACKEND}.so /data/local/tmp/saver
$ adb push ${QNN_SDK_ROOT}/examples/Saver/$QNN_TARGET_ARCH/saver_output_${QNN_BACKEND} /data/local/tmp/saver
$ adb push ./saver_output/params.bin /data/local/tmp/saver

Set LD_LIBRARY_PATH on device and execute

$ adb shell
$ cd /data/local/tmp/saver
$ export LD_LIBRARY_PATH=/data/local/tmp/saver:$LD_LIBRARY_PATH
$ ./saver_output_${QNN_BACKEND} [-l <verbose,debug,info,warn,error>] # optionally enable logging

Replay on QNN DSP or QNN HTP Backend¶

Make a directory on device, push QNN backend stub/skel, saver_output.c executable, and params.bin:

$ export QNN_BACKEND=<backend>  # where <backend> is QnnDsp or QnnHtp
$ export HEXAGON_ARCH=<v65,v66,v68,v69>  # based on device
$ adb shell "mkdir data/local/tmp/saver"
$ adb push ${QNN_SDK_ROOT}/target/$QNN_TARGET_ARCH/lib/lib${QNN_BACKEND}.so /data/local/tmp/saver
$ adb push ${QNN_SDK_ROOT}/target/$QNN_TARGET_ARCH/lib/lib${QNN_BACKEND}${HEXAGON_ARCH^}Stub.so /data/local/tmp/saver
$ adb push ${QNN_SDK_ROOT}/target/hexagon-${HEXAGON_ARCH}/lib/unsigned/lib${QNN_BACKEND}${HEXAGON_ARCH^}Skel.so /data/local/tmp/saver
$ adb push ${QNN_SDK_ROOT}/target/$QNN_TARGET_ARCH/lib/libQnnHtpPrepare.so /data/local/tmp/saver  # if QNN_BACKEND=QnnHtp
$ adb push ${QNN_SDK_ROOT}/examples/Saver/$QNN_TARGET_ARCH/saver_output_${QNN_BACKEND} /data/local/tmp/saver
$ adb push ${QNN_SDK_ROOT}/examples/Saver/params.bin /data/local/tmp/saver

Set LD_LIBRARY_PATH and ADSP_LIBRARY_PATH on device and execute:

$ adb shell
$ cd /data/local/tmp/saver
$ export LD_LIBRARY_PATH=/data/local/tmp/saver:$LD_LIBRARY_PATH
$ export ADSP_LIBRARY_PATH=/data/local/tmp/saver:$ADSP_LIBRARY_PATH
$ ./saver_output_${QNN_BACKEND} [-l <verbose,debug,info,warn,error>] # optionally enable logging
