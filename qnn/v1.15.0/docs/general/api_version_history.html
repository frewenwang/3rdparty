API Updates

1.10.0

1.10.0

January 12, 2022

API Updates

  • None.

OpDef Updates

  • DepthToSpace: Added optional mode param.

1.9.0

1.9.0

December 24, 2021

API Updates

  • QnnSystemContext: Introduce reflection API that provides info about contents of QNN context binaries, which can be utilized during execution of pre-cached contexts.

  • QnnLog: Promote log level VERBOSE ahead of DEBUG. DEBUG logs are now reserved for internal debugging, and setting level to DEBUG provides no more info than VERBOSE logs.

  • QnnCommon: Add new error error code QNN_COMMON_ERROR_SYSTEM_COMMUNICATION to indicate HW-related failures that render all contexts and graphs invalid. Apps will require resetting QNN objects constructed thus far to resume proper functioning.

OpDef Updates

  • New Ops: ScatterND

  • ElementWiseRSqrt: Clarify handling of negative elements in that the op behavior is undefined.

1.8.0

1.8.0

November 08, 2021

API Updates

  • QnnGraph: Clarify that nodes must be added in dependency order.

  • QnnGraph: Added new error code in QnnGraph_addNode to report dependency order enforcement

  • QnnGraph: Clarify that tensor IDs must be unique in the context the tensor is created.

  • QnnGraph: Clarify that tensor IDs used in an offline-prepared context binary must match to those created at runtime for execution

OpDef Updates

  • Matmul: Allow broadcasting on input operands.

  • New Ops: NV12toRGB

  • ElementWise: Clarify handling of tensors with different ranks for ElementWise ops.

  • Convert: Remove Quantization constraints on convert op.

  • FullyConnected: Added optional keep_dims param.

1.7.0

1.7.2

October 31, 2021

API Updates

  • QnnGraph: Introduce new error code in QnnGraph_execute to report invalid tensor inputs.

OpDef Updates

  • ALL: Deprecate use of notation ‘any datatype’ for tensors used in some ops. Replaced with ‘backend specific’ to indicate that BEs decide what they support.

  • StridedSlice: Update begin in ‘ranges’ param to be strictly non-negative.

  • ElementwiseSquaredDifference: Add new op for element-wise squared difference.

  • BatchNorm: Relax op def to permit N-dimensional inputs, subject to BE support.

  • InstanceNorm: Relax op def to permit N-dimensional inputs, subject to BE support.

1.5.0

1.6.0

September 30, 2021

API Updates

  • ALL: Direct access to QNN component API is now suppressed. Users have to access them strictly through QnnInterface.

  • QnnMem: Introducing component header with APIs to register externally allocated memory.

OpDef Updates

  • StridedSlice: Capture expectations in corner cases; describe hierarchy of applying params

  • StridedSlice: Fix bug with validation when end range is negative.

  • InstanceNorm: Add new optional param ‘normalized_variance’.

  • ElementwiseCos: Add new op for element-wise cosine.

1.4.1

1.5.0

August 31, 2021

API Updates

  • QnnGraph: Clarify that graphs with zero nodes are invalid, and will fail finalize step.

  • QnnGraph: Clarify that graphs cannot have optional input or output tensors.

OpDef Updates

  • LSTM: Clarify that peephole weights are specified as 1D vector because they are essentially diagonal matrices.

1.4.0

1.4.2

July 21, 2021

API Updates

  • QnnCommon: QnnSdkBuild.h, which was previously included in QnnCommon.h has been moved out

OpDef Updates

  • None

1.3.0

1.3.0

June 30, 2021

API Updates

  • QnnOpPackage: Remove support for QnnOpPackage_ImplementationV1_3_t. Custom op packages created with older versions need to upgrade to V1.4.

  • QnnTypes: Add new enum for expressing precision types.

  • QnnProfile: Remove support for deprecated profile types. Use corresponding ‘units’.

  • QnnProfile: Add new profiling unit: COUNT

OpDef Updates

  • DetectionOutput: Clarify that o/p box coordinates are normalized to [0, 1] and not absolute coordinates.

  • StridedSlice: Add new param ‘new_axes_mask’.

  • Transpose: Allow op to have rank-1 inputs.

  • Slice: Fix range for ‘split_index’ to go up to rank of input tensor.

  • L2Norm: Add vector equivalent of axis as new param called ‘axes’.

  • Convert: Allow mixed precision modes by accepting i/p and o/p tensors with equal or proportional scales and offsets.

1.2.0

1.2.0

May 31, 2021

API Updates

  • QnnGraph: Clarified expectations on several API arguments and usage. Execute API can handle null inputs for fully static graphs. Execute API matches tensor info through ID and uses previously provided info. Graphs can contain ‘unconsumed tensors’ that don’t feed into any other node.

OpDef Updates

  • ConvTranspose: Add new param output_padding to op definition.

  • ElementwiseExp: Add data type extensions to support fixed point runtimes.

  • New Ops: ElementWiseFloorDiv, ElementWiseFloorRound, ElementWiseFloorCeil, Gelu, UnPack, OneHot, LayerNorm

1.1.0

1.1.0

April 30, 2021

API Updates

  • Renamed Qnn_NotifyFunc_t to Qnn_NotifyFn_t (for naming compliance). Considered backwards compatible as this type is not used/supported yet.

OpDef Updates

  • Topk: Added support for INT32 datatype in indices output.

  • Correlation1D: Fixed output tensor shape definition. No change in BE behavior.

1.0.0

1.0.0

April 02, 2021

API Updates

  • QnnInterface: Introduce concept of exposing QNN APIs through ‘providers’.

  • Backends register their API interface as provider for users to access instead of direct access to direct APIs that shall be deprecated in a future release.

  • All APIs: Mark raw usage as deprecated; users need to access them through QnnInterface

  • All APIs: Introduce helper macros for safe initialization of QNN data structures.

  • QnnError.h: Remove this header from SDK, though it can still be used internally

  • QnnCommon: Introduce error codes reserved for saver tool

  • QnnProperty: Remove deprecated property keys from API header

  • QnnGraph: Add caveats to tensor handling in graph execute

  • QnnTypes: Add clarity to usage of error codes in API error handles

  • QnnOpPackage.h: Simplify packageInfo in QnnOpPackage_Info_t to be a single pointer

OpDef Updates

  • Gather: Change data type for indices to INT32 to match source framework definitions

  • DetectionOutput: Fix dims of anchor positions to include batch

  • LSTM: Enforce data dype for LSTM params to FLOAT32

  • DepthWiseConv2D: Update def to state that channel_out can be a multiple of channel_in

  • Pack: Axis param can have max value equal to rank of input tensor.
