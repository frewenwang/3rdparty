Introduced in QNN SDK Version

Type of Change

Description

1.1.0

1.8.0

API Updates

  • QnnOpPackage: Remove support for QnnOpPackage_ImplementationV1_3_t.

  • Custom op packages created with older versions need to upgrade to V1.4.

OpDef Updates

  • MatMul: Added support for default value for broadcast_dims parameter

  • Transform: Added support for QNN_DATATYPE_INT_32 data type

  • Cast: Added support for QNN_DATATYPE_UINT_8 and QNN_DATATYPE_INT_32 data types

  • StridedSlice: Added support for QNN_DATATYPE_INT_32, QNN_DATATYPE_UINT_32,

    QNN_DATATYPE_INT_8 and QNN_DATATYPE_UINT_8 data types

  • OneHot: Added support for QNN_DATATYPE_INT_32 data type for Input

  • Support added for ElementWiseCeil Op

  • Concat: Added support for QNN_DATATYPE_UINT_32 data type

  • ElementWiseMultiply: Added support for QNN_DATATYPE_UINT_32 data type

  • Support added for OneHot Op

  • AvgPool: Added support for count_pad_for_edges parameter

  • Support added for Unpack Op

  • ArgMin: Added support for keep_dim parameter

  • Support added for FloorDiv Op
