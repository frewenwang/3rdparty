Setup¶

This page will outline the setup required for using the QNN SDK. The page is split into Platform Dependencies and Environment Setup.

Platform Dependencies¶

Host OS¶

The host operating system that the QNN SDK is verified with is Ubuntu 18.04 LTS (Bionic).

Python¶

QNN SDK distributions are supported on python3 only. Download and Installation instructions vary based on the host OS, and for Linux distributions that do not come pre-installed with python3 you may install it with the following commands:

$ sudo apt-get update
$ sudo apt-get install python3=3.6.9

Note

For older Linux distributions that may not contain python3 in the base image download from a Personal Package Archive (PPA).

Example command:

$ sudo add-apt-repository ppa:your_ppa_dist

PIP:

If it isn’t present pip3 needs to be installed on your system in order to allow further installations and management of python packages.

$ sudo apt-get install python3-pip

Some additional python3 packages need to be available in your environment in order to be able to interact with QNN modules and tools. This QNN release is verified to work with the versions of the packages specified below:

Run the following script to check and install missing dependencies:

$ source ${QNN_SDK_ROOT}/target/x86_64-linux-clang/bin/check-python-dependency.sh

Linux¶

Compiling for the HTP v68 emulator on X86 requires clang++. This QNN SDK release is verified to work with clang-9.0.0.

Run the following script to check and install missing linux dependencies:

$ source ${QNN_SDK_ROOT}/target/x86_64-linux-clang/bin/check-linux-dependency.sh

Note

To set QNN_SDK_ROOT see Environment Setup.

ML Frameworks¶

In order to convert ML models trained on different frameworks into intermediate representations consumable by QNN you may need to download and install the corresponding frameworks on your host machine.

This QNN release is verified to work with versions of the ML training frameworks specified below:

  • Caffe: caffe_1.0.0-rc3 from the open-source tree at https://github.com/BVLC/caffe.

  • Caffe-SSD: A fork of the Caffe development tree available at https://github.com/weiliu89/caffe/tree/ssd.

  • Tensorflow: tf-1.15.0, or tf-2.3.0

  • TFLite: tflite-2.3.0 available at https://pypi.org/project/tflite/2.3.0/

  • PyTorch: torch-1.8.1

  • ONNX: onnx-1.6.0

Compiler Toolchains¶

The QNN SDK allows users to compile custom operation packages to be used with the different backends such as the CPU, GPU, HTP and DSP. You may need to install appropriate cross-compilation toolchains in order to compile such packages for a particular backend.

MAKE:

Operation packages are compiled with a front-end that is written with Makefiles. If make is not available on your host machine install it using the following command:

$ sudo apt-get install make

CPU:

Compiler used for X86_64 is clang 9.0.0. ARM CPU targets are built using Android NDK. The current Android NDK used is Android NDK 19c, which uses clang 8.02.

GPU:

GPU kernels are written based on OpenCL. You should implement operation packages for the GPU based on OpenCL headers with minimum version OpenCL1.2.

HTP/DSP:

Compiling for HTP/DSP requires the use of the Hexagon toolchain available from Hexagon-SDK. Versions of the SDK are available at https://developer.qualcomm.com/software/hexagon-dsp-sdk/tools.

This QNN release is verified to work with:

  • HTP: Hexagon SDK version 4.2.0, Hexagon-sdkTools version: 8.4.09

  • DSP: v65 - Hexagon SDK version 3.5.2, Hexagon-sdkTools version: 8.3.07, v66 - Hexagon SDK version 4.1.0, Hexagon-sdkTools version: 8.4.06

  • Additionally, compiling for HTP/DSP requires clang++.

Further setup instructions are available at $HEXAGON_SDK_ROOT/docs/readme.html, where HEXAGON_SDK_ROOT is the location of the Hexagon-SDK installation.

Note

Hexagon-sdkTools version 8.4.09/8.4.06/8.3.07 is not currently pre-packaged into Hexagon SDK version 4.2.0/4.1.0/3.5.2. It needs to be downloaded separately and placed at the location $HEXAGON_SDK_ROOT/tools/HEXAGON_Tools/ .

Environment Setup¶

After platform dependencies have been satisfied, the user environment can be set with the provided envsetup.sh script.

Set ANDROID_NDK_ROOT:

$ export ANDROID_NDK_ROOT=<PATH-TO-NDK>

Note

This QNN release is verified to work with Android NDK version r19c.

Then run:

$ source ${QNN_SDK_ROOT}/target/x86_64-linux-clang/bin/envsetup.sh

This will set/update the following environment variables:

  • QNN_SDK_ROOT

  • PYTHONPATH

  • PATH

  • LD_LIBRARY_PATH

The QNN API headers are in ${QNN_SDK_ROOT}/include.

The tools are in ${QNN_SDK_ROOT}/target/x86_64-linux-clang.

Target specific backends are in ${QNN_SDK_ROOT}/target/*/lib.

TensorFlow Setup¶

To setup environment for TensorFlow run:

$ source ${QNN_SDK_ROOT}/target/x86_64-linux-clang/bin/envsetup.sh -t <your_tensorflow_installation_path>

Caffe Setup¶

To setup environment for Caffe run:

$ source ${QNN_SDK_ROOT}/target/x86_64-linux-clang/bin/envsetup.sh -c <your_caffe_installation_path>

ONNX Setup¶

To setup environment for ONNX run:

$ source ${QNN_SDK_ROOT}/target/x86_64-linux-clang/bin/envsetup.sh -o <your_onnx_installation_path>

TFLite Setup¶

Please install TFLite as a standalone Python module using https://pypi.org/project/tflite/2.3.0/ and ensure TFLite is in your PYTHONPATH by using this test code:

$ python -c "import tflite"

PyTorch Setup¶

Please install PyTorch as a standalone Python module using https://pytorch.org/get-started/locally v1.8.1 and ensure PyTorch is in your PYTHONPATH by using this test code:

$ python -c "import torch"
