## 核心组件

### 1. 构建系统 (`compiler/`)
- **`build.sh`**: 优化的主构建脚本，具有增强的错误处理
- **`cmake/`**: 平台特定工具链配置
- **`cmake_args/`**: 不同平台的优化 CMake 配置
- **`scripts/`**: 特定用例的参考构建脚本

### 2. 配置文件 (`compiler/cmake_args/`)
- **`android_options.txt`**: 针对 Android 移动平台优化
- **`host_options.txt`**: macOS/Linux 开发的完整功能集
- **`embedded_options.txt`**: 嵌入式平台的最小配置
- **`common_options.txt`**: 所有平台共享的选项

### 3. 文档 (`compiler/docs/`)
- 不同平台和用例的综合指南
- 故障排除和最佳实践
- 示例配置和脚本

### 4. 构建输出
- **`build/`**: 中间构建文件（自动生成）
- **`lib/`**: 按版本和平台组织的最终编译库

## 使用示例

### 构建 Android ARM64
```bash
cd compiler
./build.sh -t 2 -r -n opencv -v 4.11.0
```

### 构建当前平台
```bash
cd compiler
./build.sh -t 0 -r
```

### 使用自定义选项构建
```bash
cd compiler
./build.sh -t 2 -r -a "-DWITH_CUDA=ON -DBUILD_opencv_dnn=ON"
```

## 优化特性

1. **平台特定配置**: 每个平台都有优化的 CMake 设置
2. **错误处理**: 全面的错误检查和有意义的错误消息
3. **日志记录**: 具有不同严重级别的结构化日志
4. **模块化设计**: 易于扩展新平台
5. **文档**: 综合指南和示例
6. **备份系统**: 保留原始脚本供参考

## 与 FantasyCXX 集成

根目录中的 `opencv.cmake` 文件提供了与主 FantasyCXX 项目构建系统的集成。

## 版本管理

- `lib/` 目录中可以共存多个 OpenCV 版本
- 按版本号组织特定版本的构建
- 易于在不同 OpenCV 版本之间切换
