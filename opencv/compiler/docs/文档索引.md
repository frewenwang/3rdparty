## 🔍 按功能分类

### 构建配置
- 平台选择：使用 `-t` 参数指定目标平台
- 构建类型：`-r` (release), `-d` (debug), `--RelWithDebInfo`
- 自定义参数：使用 `-a` 参数传递额外的 CMake 选项

### 平台支持
- **Android**: ARM64, ARMv7, x86, x86_64
- **iOS**: ARM64, ARMv7
- **QNX**: ARMv7, ARM64
- **自定义平台**: AX620A
- **宿主平台**: macOS, Linux

### 优化选项
- **体积优化**: 使用嵌入式配置，禁用不需要的模块
- **性能优化**: 启用硬件加速，并行处理
- **开发优化**: 启用完整功能集，测试和示例

## 📝 常用命令参考

```bash
# 显示帮助信息
./build.sh --help

# 列出所有支持的目标平台
./build.sh --list-targets

# 构建当前平台 Release 版本
./build.sh -t 0 -r

# 构建 Android ARM64
./build.sh -t 2 -r -n opencv -v 4.11.0

# 构建 Android ARMv7 Debug 版本
./build.sh -t 1 -d -n opencv -v 4.11.0

# 自定义构建参数
./build.sh -t 2 -r -a "-DWITH_CUDA=ON -DBUILD_opencv_dnn=ON"

# 指定安装路径
./build.sh -t 2 -r -i /opt/opencv/android
```

## 🐛 故障排除流程

1. **检查前置条件**
   - 确保安装了必需的工具（CMake, Make）
   - 验证环境变量设置（Android NDK 等）

2. **查看错误信息**
   - 阅读详细的错误输出
   - 检查 [TROUBLESHOOTING.md](compiler/docs/TROUBLESHOOTING.md)

3. **验证配置**
   - 检查目标平台索引是否正确
   - 验证 CMake 参数格式
   - 确认工具链文件存在

4. **清理重试**
   - 删除构建目录进行干净构建
   - 检查磁盘空间是否足够
